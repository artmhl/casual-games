# 🤖 Інструкція для ШІ: Як додати нову гру

Цей документ — повна інструкція для додавання нової гри у портал.  
**Попередній контекст не потрібен.** Просто читай цей файл.

---

## Структура проєкту

```
src/
  games/
    registry.js          ← ЄДИНИЙ файл, де реєструються ігри
    <slug>/
      index.jsx          ← Вся логіка і UI гри в одному файлі
  components/
    HomePage.jsx         ← Список ігор (не чіпати)
    GamePage.jsx         ← Обгортка гри (не чіпати)
    LoadingScreen.jsx    ← Не чіпати
    NotFound.jsx         ← Не чіпати
  index.css              ← Глобальні стилі + CSS-змінні
  App.jsx                ← Роутинг (не чіпати)
```

---

## Кроки для додавання нової гри

### Крок 1 — Створи файл гри

Шлях: `src/games/<slug>/index.jsx`

**Slug** — латинські літери та дефіс, наприклад: `spy`, `truth-or-dare`, `word-chain`

### Крок 2 — Зареєструй гру в реєстрі

Відкрий `src/games/registry.js` і додай запис в масив `games`:

```js
{
  slug: "spy",               // має збігатись з назвою папки
  title: "Шпигун",           // назва українською
  description: "Знайди шпигуна серед гравців!", // 1 речення
  emoji: "🕵️",              // один емодзі
  minPlayers: 3,             // мін. гравців (або видали поле)
  maxPlayers: 12,            // макс. гравців (або видали поле)
  tags: ["компанія", "блеф"], // 1–3 коротких теги
  component: lazy(() => import("./spy/index.jsx")),
}
```

Не забудь імпортувати `lazy` з react — він вже є на початку файлу.

## Шаблон файлу гри

```jsx
// src/games/<slug>/index.jsx
// Гра повністю автономна. Не імпортує нічого із сусідніх ігор.

import { useState } from "react";

// CSS-змінні доступні глобально з index.css:
// --bg, --bg2, --bg3, --accent, --accent2, --text, --text2
// --radius, --radius-sm

export default function MyGame() {
  // Весь стан гри тут
  const [phase, setPhase] = useState("setup"); // "setup" | "play" | "result"

  return (
    <div style={{ display: "flex", flexDirection: "column", gap: 16 }}>
      {phase === "setup" && <Setup onStart={() => setPhase("play")} />}
      {phase === "play" && <Play onEnd={() => setPhase("result")} />}
      {phase === "result" && <Result onRestart={() => setPhase("setup")} />}
    </div>
  );
}
```

## CSS-змінні (доступні в будь-якій грі)

| Змінна | Значення | Призначення |
|---|---|---|
| `--bg` | `#0f0f13` | Основний фон |
| `--bg2` | `#1a1a24` | Картки, панелі |
| `--bg3` | `#23233a` | Бордери, чіпи |
| `--accent` | `#7c6af7` | Фіолетовий акцент |
| `--accent2` | `#a78bfa` | Світліший акцент |
| `--text` | `#f0eeff` | Основний текст |
| `--text2` | `#9b9ab8` | Другорядний текст |
| `--radius` | `16px` | Великий радіус |
| `--radius-sm` | `10px` | Малий радіус |

## Готові CSS-класи (з index.css)

- `.btn-primary` — фіолетова кнопка, width: 100%
- `.btn-secondary` — темна кнопка з бордером, width: 100%

## Правила написання ігор

1. **Один файл** — вся гра в `index.jsx`, без сабфайлів
2. **Нема зовнішніх залежностей** — тільки React (useState, useEffect тощо)
3. **Мобільна оптимізація** — великі кнопки (мін. 48px), зручне торкання
4. **Без навігації** — кнопка "Назад" вже є у GamePage
5. **Inline стилі або CSS-змінні** — не створюй окремих .css файлів для гри
6. **Повний цикл** — гра має: екран налаштування → ігровий процес → результат/перезапуск
